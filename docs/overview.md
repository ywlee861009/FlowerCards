# FlowerCards — 제품 개요 (Overview)

> 프로젝트의 나침반. 모든 세부 문서(룰/디자인/아키텍처)는 이 문서의 방향을 따른다.
> 최종 갱신: 2026-07-07

## 1. 한 줄 정의
캔버스로 직접 그린, **손맛과 사운드**에 집중한 안드로이드 **2인 맞고(고스톱)** 게임.

## 2. 핵심 가치 (Pillars)
이 프로젝트의 모든 의사결정은 아래 두 축을 최우선 기준으로 판단한다.

1. **손맛 (Tactile Game Feel)**
   - 카드를 집고 내려놓고 뒤집는 물리적 감각, 즉각적인 입력 반응, 획득 시의 타격감.
   - 애니메이션 커브·햅틱·카메라 반응이 하나의 "촉감"으로 느껴져야 한다.
2. **사운드 (Sound)**
   - 카드 슬라이드, 매칭 "탁", 뻑, 고 선언, 승패 등 상황별 SFX가 저지연으로 손맛과 동기화된다.
   - 사운드는 시각 피드백을 증폭시키는 핵심 요소이지, 배경 장식이 아니다.

## 3. 타깃 스코프 (초기 결정)
| 항목 | 결정 | 비고 |
|---|---|---|
| 게임 모드 | **2인 맞고** | 각 10장, 바닥 8장, 더미 20장 |
| 대전 상대 | **AI(싱글플레이) 우선** | 온라인 멀티는 초기 스코프 제외 |
| 플랫폼 | Android | 최소 SDK는 아키텍처 문서에서 확정 |
| 네트워크 | 없음(오프라인) | 광고/결제 등은 MVP 이후 논의 |

## 4. MVP 범위
**포함**
- 화투 48장 / 2인 맞고 1판 완결 루프 (딜 → 턴 → 매칭 → 점수 → 고/스톱 → 결과)
- 표준 점수 규칙 (광/열끗/띠/피, 고, 피박·광박, 뻑·따닥·쪽·싹쓸이 등)
- AI 상대 (기본 난이도 1종)
- 손맛 3종 세트: 카드 드래그/스냅, 뒤집기, 획득 연출 + 대응 SFX + 햅틱
- 결과 화면 + 재시작

**제외 (MVP 이후)**
- 온라인 멀티플레이, 랭킹, 계정
- 상점/스킨/결제, 광고
- 3인 고스톱, 지역 룰 변형 다수
- 튜토리얼(간단 안내만)

## 5. 성공 기준 (Definition of "느낌 좋다")
- 카드 터치 → 시각 반응까지 체감 지연이 없다(입력 프레임 내 반응).
- 매칭·획득 순간 사운드가 어긋나지 않고 "짝" 붙는다.
- 스크롤/애니메이션 중 프레임 드랍이 눈에 띄지 않는다.
- 한 판을 끝내고 "한 판 더" 누르고 싶다.
