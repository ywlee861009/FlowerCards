# 게임 루프 / 화면 흐름 (Game Loop & Screen Flow)

> 화면 구성의 **소스 오브 트루스**. `overview.md`의 방향(손맛·사운드 우선)을 따르며, 룰은 `rules.md`, 점수는 `scoring.md`를 기준으로 한다.
> 렌더링 방식(SurfaceView vs Compose Canvas)은 미결정이므로 이 문서는 **렌더링에 독립적인 레이아웃/흐름 수준**으로 기술한다.
> 최종 갱신: 2026-07-08 · 대상: 2인 맞고, AI 싱글플레이, **세로(Portrait) 고정**

## 0. 확정된 설계 결정 (사용자 승인 완료)

| # | 항목 | 결정 |
|---|---|---|
| 1 | 화면 방향 | **세로(Portrait) 고정** |
| 2 | 획득패 처리 | **압축 스트립** (카테고리별 겹침 스택 + 개수 배지). 남는 세로 예산을 바닥패·손패에 몰아줌 |
| 3 | 내 손패 배치 | **A. 미세 아크 겹침 1줄** (선택 시 팝업+확대, 터치 히트박스 확장) |
| 4 | 턴 타이머 | **없음** (플레이어 입력까지 대기) |
| 5 | 흔들기/폭탄 입력 | **액션바 버튼** (조건 충족 시 활성) |

### 설계 대전제 — 세로 예산 경쟁
세로 화면 맞고의 최대 난제는 **세로 예산(vertical budget) 경쟁**이다. 위→아래로 밴드가 8개 필요하다:

`AI HUD → AI 획득패 → AI 손패 → 바닥패 → 더미 → 내 획득패 → 내 손패 → 액션바`

이걸 다 제값으로 넣으면 게임의 심장인 **바닥패(8장+뻑 더미)가 짓눌린다**. 따라서:
- 획득패(양쪽)는 **압축 스트립**으로 처리한다.
- **바닥패와 내 손패에 세로 예산을 몰아준다** (이 둘이 손맛의 90%).
- AI 손패는 뒷면이므로 최소 높이만 준다.
- 특수상황·고/스톱은 **별도 화면이 아니라 보드 위 오버레이/레이어**로 처리한다 → 공간 연속성이 곧 손맛.

## 1. 화면(스크린) 목록

| 화면 | 목적 |
|---|---|
| **Splash / 부팅** | 로고 + 에셋/아틀라스 프리로드(첫 프레임 비트맵 디코딩 지연 숨김) |
| **타이틀 / 메인** | 게임 시작 진입점. "시작" 한 방에 손맛/사운드 온보딩(탭 시 카드 착 소리) |
| **인게임 보드** | 1판 완결 루프의 무대. 딜~점수~고/스톱이 전부 여기서 벌어짐 |
| **일시정지 오버레이** | 보드 위 반투명. 계속 / 재시작 / 사운드 토글 / 나가기 |
| **설정 / 라이선스** | 사운드·햅틱·카드 스킨(뒷면)·애니 속도, 오픈소스 라이선스. 타이틀/일시정지/액션바에서 진입 |
| **결과 화면** | 승패 판정 + 점수 내역(광/띠/열끗/피)의 카타르시스 연출 + 재시작/종료 |
| **특수상황 연출 레이어** | 뻑·쪽·따닥·싹쓸이·고 선언 등을 보드 위에 얹는 **비파괴** 연출 레이어 |

> 튜토리얼은 별도 화면이 아니라 **첫 플레이 시 보드 위 코치마크(1회성 툴팁)**로 처리한다. 별도 튜토리얼 화면은 MVP 범위 초과(`overview.md` §4).

## 2. 화면 전환 흐름도

```
[Splash/프리로드]
      │ (에셋 로드 완료)
      ▼
[타이틀/메인] ──────────────► [설정 오버레이]
      │  ▲                         │
"게임 시작"│  └──────(뒤로)──────────┘
      ▼
[인게임 보드]
      │
      ├─(딜 애니) ─► [턴 루프] ◄──────────┐
      │                │                  │
      │        내 턴 / AI 턴 교대          │
      │                │                  │
      │        (특수상황 발생 시)          │
      │                ▼                  │
      │      [특수상황 연출 레이어]────────┘ (연출 후 턴 복귀)
      │                │
      │        (점수 3점↑ 도달, rules §5)
      │                ▼
      │        [고/스톱 오버레이]
      │            │        │
      │          "고"     "스톱"
      │            │        │
      │        턴 루프로     ▼
      │        복귀 ──►  [결과 화면]
      │
      ├─(총통, rules §3) ──────► [결과 화면(즉시 종료)]
      ├─(더미 소진, 승자 없음)─► "나가리" 연출 ─► [결과 화면(무승부)]
      │
      │◄────── ⏸ 아무 때나 ──► [일시정지 오버레이]
      │                            ├─ 계속 ─► 보드 복귀
      │                            ├─ 재시작 ─► 보드(새 딜)
      │                            └─ 나가기 ─► [타이틀]
      ▼
[결과 화면]
      ├─ "다시 하기" ─► [인게임 보드](새 딜)
      └─ "나가기" ─────► [타이틀]
```

**핵심**: 특수상황과 고/스톱은 화면을 갈아끼우지 않는다. 보드 위 오버레이/레이어로 얹어 공간의 연속성(=손맛)을 지킨다.

**판 종료 조건 3종**(rules §6)은 모두 결과 화면으로 수렴한다: ① 3점↑에서 스톱, ② 총통(즉시), ③ 나가리(더미 소진).

**현재 구현 메모(2026-07-08)**: 설정은 우선 `:feature:setting`의 독립 Compose 화면으로 구현했다. 게임 화면 placeholder의 [설정] 버튼에서 진입하고 [완료]로 돌아온다. 보드 UI가 완성되면 이 화면을 일시정지/타이틀/액션바 진입 흐름에 맞춰 오버레이 또는 전체 화면 중 하나로 최종 확정한다.

## 3. 인게임 보드 세로 레이아웃 구획

세로 화면 위→아래. 오른쪽 괄호는 대략의 세로 높이 배분 제안이다. 딜 배치(손10/손10/바닥8/더미20, rules §3)와 정합한다.

```
┌───────────────────────────────────────┐
│ [≡]        AI ●  남은패:10       [🔊]  │  ← HUD 상단 (≈6%)
│  일시정지          AI 상태               │
├───────────────────────────────────────┤
│  AI 획득패 (압축 스트립)                 │  ← AI 획득 (≈9%)
│  [광×2][띠×3][열×1][피×5]  점수:3       │
├───────────────────────────────────────┤
│      🂠 🂠 🂠 🂠 🂠 🂠 🂠 🂠 🂠 🂠         │  ← AI 손패(뒷면) (≈8%)
├═══════════════════════════════════════┤
│                                        │
│        [ 바닥패 8장 + 뻑 더미 ]         │
│        🀫  🀫  🀫  🀫                    │  ← 바닥패 (≈24%)
│        🀫  🀫  🀫  🀫                    │     (게임의 심장)
│                  ┌───┐                  │
│                  │더미│ 20  ← 중앙 우측  │
│                  └───┘                  │
├───────────────────────────────────────┤
│  내 획득패 (압축 스트립)                 │  ← 내 획득 (≈10%)
│  [광×1][띠×2][열×3][피×7]  점수:6 ▲     │
├───────────────────────────────────────┤
│                                        │
│    🎴 🎴 🎴 🎴 🎴 🎴 🎴 🎴 🎴 🎴        │  ← 내 손패 (≈20%)
│    (미세 아크 겹침 1줄, 선택 시 팝업)     │     (손맛의 심장)
│                                        │
├───────────────────────────────────────┤
│  [흔들기]   [폭탄]        [설정]         │  ← 액션바 (≈7%)
└───────────────────────────────────────┘
```

각 영역이 담는 것:

- **HUD 상단**: 일시정지 버튼, AI 아바타/난이도, AI 남은 손패 수, 사운드 토글. 고 선언 누적(●●)도 여기 또는 획득 스트립 옆에 표시.
- **AI 획득패 스트립**: 광/띠/열끗/피 카테고리별 겹침 스택 + 개수 배지 + AI 현재 점수. 획득 순간 카드가 여기로 날아가 꽂힘.
- **AI 손패**: 뒷면 10장, 겹침. AI가 낼 때 한 장이 바닥으로 이동. 정보 표시용이라 최소 높이.
- **바닥패(중앙, 최대 예산)**: 8장 그리드 + 진행 중 생긴 월별 더미/뻑 더미. 매칭 조준·충돌·뒤집기가 여기서 벌어짐. **더미(덱)는 바닥 영역 안 우측**에 배치해 "손패→바닥→더미 뒤집기" 시선 흐름을 한 영역에서 완결.
- **내 획득패 스트립**: AI와 대칭. 점수 옆 **▲(고 가능 임계 3점 도달)** 표식.
- **내 손패(하단, 2순위 예산)**: 10장. §4·§6 참조.
- **액션바**: 흔들기/폭탄은 조건 충족 시에만 활성(평소 회색 비활성). 설정 진입.

## 4. 핵심 인터랙션 / 상태 오버레이

### 4-1. 고 / 스톱 (3점↑ 도달 시, rules §5 · scoring §3)
- **위치**: 화면 중앙~중앙하단. 바닥패를 살짝 dim 처리하고 그 위에 카드형 모달.
- **구성**: 현재 점수 + 확정 획득 미리보기, [고] / [스톱] 큰 버튼 2개. 고 누적 횟수(1·2고 +1점, 3고↑ ×2 배수, scoring §3) 표시.
- **손맛**: 버튼 살짝 pulse. "고" 선택 시 모달이 위로 터지듯 사라지며 화면 가장자리 골드 글로우.
- 턴 타이머는 두지 않는다(결정 #4) → 플레이어 입력까지 대기.

### 4-2. 흔들기 / 폭탄 선언 (rules §4.7) — 액션바 버튼 방식(결정 #5)
- **트리거**: 손패에서 같은 월 3장 감지 시 액션바 [흔들기]/[폭탄] 활성 + 해당 3장 손패에서 살짝 들림(하이라이트).
- **흔들기**: 카드 내기 **전** 버튼 선언 → 확인 미니 팝업 → 3장 좌우 흔들림 모션 + 사운드. 해당 판 점수 ×2.
- **폭탄**: 버튼 선언 후 3장을 바닥에 내리꽂는 슬램 → 4장 획득, 점수 ×2, 추가 턴(추가 턴 여부는 rules ⚠️#6 결정 대기).

### 4-3. 특수상황 연출 (뻑/자뻑/따닥/따조/쪽/싹쓸이/나가리) — 비파괴 오버레이
원칙: 화면 전체를 덮지 않고 **발생 지점(바닥 위)**에 스탬프/버스트를 얹는다. 게임 상태가 계속 보여야 손맛이 산다.

- **뻑/자뻑**(rules §4.2·§4.3): 바닥 해당 월 더미 위에 "뻑" 스탬프 + 짧은 화면 흔들림 + 붉은 플래시. 자뻑은 스탬프 색/사운드 차별화. → 판 상태에 영향 주므로 **바닥 위 지속 스탬프**.
- **쪽**(rules §4.5): 더미에서 뒤집힌 카드가 방금 낸 카드와 맞을 때, 그 지점에 "쪽" 말풍선 + 상대 피 1장이 내 스트립으로 날아옴.
- **따닥/따조**(rules §4.4): 4번째 매칭 순간 캡처 지점에서 별 버스트 + "따닥" 팝. 상대 피 뺏김 이동.
- **싹쓸이**(rules §4.6): 바닥이 비는 순간 가로 스윕 라이트 + "싹쓸이" 얇은 배너, 상대 피 1장 이동.
- **나가리**(rules §4.8): 더미 소진 & 승자 없음 → 중앙 반투명 "나가리" + 다음 판 배수 표식(rules ⚠️#7) 후 결과 화면.

> 구분 규칙: **판 상태에 영향 주는 것(뻑) = 바닥 위 지속 스탬프**, **순간 이벤트(쪽/따닥) = 짧은 토스트/버스트**.

## 5. 전환·상황별 손맛/사운드 트리거 매핑

각 항목은 **입력 → 시각 반응 → 사운드 트리거 지점** 세트다. 실제 duration/easing 수치는 별도 game-feel/sound-design 문서에서 확정하고, 여기선 **트리거 지점**만 정의한다. 사운드는 시각의 "임팩트 프레임"에 정확히 물린다.

| 상황 | 입력 | 시각 반응 | 사운드 트리거 지점 |
|---|---|---|---|
| 딜 | 판 시작 | 더미→손패/바닥 순차 비산(stagger), 착지 시 미세 반동 | **각 카드 착지 프레임마다** "촥촥"(연속) |
| 카드 집기 | 손패 카드 터치다운 | 해당 카드 위로 팝업 + 스쿼시 | 터치다운 즉시 "톡" |
| 카드 드래그 | 이동 | 손가락 추종 + 살짝 기울기 | (무음, 이동감은 시각만) |
| 바닥에 냄 | 릴리스/스냅 | 바닥 슬롯 스냅 + 충돌 반동(squash) | **스냅 착지 프레임** "탁" |
| 매칭 성공 | 같은 월 겹침 | 두 카드 붙으며 튐 후 획득 스트립으로 비행 | 겹침 프레임 "찰칵" + 착지 시 "촥" |
| 더미 뒤집기 | 자동 | 3D flip(반투명 하이라이트) | **뒷→앞 전환 90° 프레임** "촤르륵" |
| 쪽 | 더미 뒤집기=낸 카드 매칭 | "쪽" 팝 + 상대 피 비행 | 매칭 확정 프레임 "쪽!" |
| 뻑/자뻑 | 매칭 실패로 더미 형성 | 붉은 플래시 + 화면 shake + 스탬프 | 스탬프 찍히는 프레임 "뻑!" |
| 따닥/따조 | 4번째 매칭 | 별 버스트 | 버스트 시작 프레임 "따닥!" |
| 싹쓸이 | 바닥 비움 | 가로 스윕 라이트 | 스윕 시작 프레임 "촤아—" |
| 흔들기 | 선언 확정 | 3장 좌우 흔들림 | 흔들림 시작 프레임 "촤라락" |
| 폭탄 | 3장 슬램 | 바닥 내리꽂기 + 큰 반동/shake | 착지 프레임 "쾅!" |
| 고 선언 | [고] 탭 | 모달 상방 폭발 + 가장자리 골드 글로우 | 탭 즉시 "고!" 보이스 |
| 스톱/승리 | [스톱] 탭 | 결과로 전환하며 획득패가 점수판으로 집결 | "스톱" + 승리 팡파레(결과 진입) |
| 나가리 | 더미 소진 | 중앙 "나가리" 페이드 | 페이드인 프레임 "나가리" |
| 화면 전환(타이틀↔보드) | 버튼 | 카드 한 장이 화면을 쓸며 전환(와이프) | 전환 시작 "촥" |

> 이 트리거 목록은 추후 `docs/planning/sound-design.md`와 **대조 검수** 대상이다.

## 6. 손패 배치: 미세 아크 겹침(결정 #3) 상세

세로 폭 360~420dp에 카드 10장은 겹침 없이 배치 불가(600dp+ 필요)하므로 **겹침은 전제**다. 강한 부채꼴·가로 스크롤은 배제한다(조준 정확도 희생 / 10장 동시 조망 붕괴). 채택한 A안의 보완책:

- **선택 팝업**: 터치다운 즉시 해당 카드가 위로 솟고 확대 → 릴리스로 확정. 확정 전 아래로 다시 내리면 취소.
- **터치 히트박스 확장**: 시각적 겹침과 무관하게 각 카드의 실제 탭 영역을 "인접 카드 경계 중앙"까지 넓혀 보이는 폭보다 크게.
- **카드 좌측 식별 영역**: 월/광 표식이 카드 왼쪽에 오도록 아트 설계 → 겹쳐도 월 판별 가능.
- **매칭 가능 힌트(옵션)**: 내 턴에 바닥과 맞는 손패 카드에 미세 글로우. 초보 배려. 상급자용 off 옵션 제공.

## 7. 열린 질문 (다음 단계 결정)

1. **아트 방향(전통 vs 모던)**: 카드/보드 톤을 전통 화투 질감으로 갈지 모던 플랫으로 갈지. 카드 비주얼 시안 전에 결정 필요 → `docs/design/art-direction.md`.
2. **설정 표시 방식**: 최종 UX를 보드 위 오버레이로 유지할지, 현재 구현처럼 별도 화면으로 둘지.
3. **매칭 가능 힌트 기본값**: 기본 on/off 및 옵션 노출 방식.
4. **결과 화면 연출 밀도**: 점수 내역을 단계별로 카운트업할지, 한 번에 보여줄지.
5. **rules.md ⚠️ 결정 필요 8건 중 화면 영향 항목**: 흔들기/폭탄 배수 표시, 쌍피 표시 여부 등이 HUD/스트립에 반영됨 → 룰 확정 후 반영.
